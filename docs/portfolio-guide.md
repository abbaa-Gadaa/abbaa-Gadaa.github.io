## 4. Instructor Feedback Summary

### Feedback Round 1 — Core Issues:
- **"This is a document, not a portfolio"** — too text-heavy, reads like a report
- **No visuals** — no screenshots, UI demos, diagrams, interaction examples
- **Cut text by 40-50%** — keep only what proves value
- **Landing page needs a hook** — Who are you? What do you do? Why should I care?
- **Template vs Artifact** — explanation is doing the work, the artifact itself isn't shown
- **Netflix analogy** — visual options immediately, no paragraphs, you know where to click

### Feedback Round 2 — Execution Gap:
- **"Execution doesn't match reflection claims"** — said visual roadmap exists, but it doesn't
- **Still describing, not demonstrating** — needs flow diagrams, before/after, output examples
- **Weak value communication** — "Why This Matters" is generic, needs measurable impact
- **Navigation issue** — reviewer should never have to guess where the work is
- **Credibility gap** — don't claim what isn't shown
- **⚠️ Rubric warning:** "appears inauthentic (i.e. careless use of generative AI)" = fail

## 5. Portfolio Best Practices (from research)

### Structure:
- Work samples (fewer strong > many weak)
- Case studies with process explanations (problem → approach → result)
- Professional background / resume
- About page (specialization, problems you solve)
- Contact information

### Key principles:
- **Quality over quantity** — smaller set, well-explained
- **Keep it focused** — show work aligned with your direction
- **Structure projects clearly** — problem, role, approach, outcome
- **Portfolio is a product** — treat it with same excellence as the projects inside
- **Clarity over volume, decisions over deliverables**
- **Guide the reader without friction**

### The 4 purposes of a portfolio:
1. Back up your resume with tangible evidence
2. Show your professional standards and taste
3. Reveal how you think and make decisions
4. Central hub for all professional material

## 6. Design Inspiration Notes (from video research)

### Portfolio 1: Igor Terekhov (igorterekhov.com) — Painter

**What makes it great:**
- Warm cream/beige background (#FAF4EB range) — feels like gallery walls
- Minimal, editorial typography — serif for display, uppercase tracking for labels
- Massive whitespace — gallery-like breathing room
- Scroll-driven animations — paintings scale and appear as you scroll
- Full-bleed immersive photos filling the viewport
- Asymmetric layouts — intentionally scattered, not grid-locked
- Dark sections for contrast (footer, gallery photos)
- Clean horizontal dividers between sections
- Museum catalog aesthetic — metadata under each work (title, dimensions, year, medium)
- Navigation: name centered, section links right, language toggle left

**Pages observed:**
- Homepage with individual paintings revealed on scroll
- "Periods" page — creative periods as numbered date ranges with organic image grids
- "Biography" page — B&W portrait, timeline bio, exhibition history
- Footer with contacts, technique info, credits

**Design takeaways to use:**
- Warm color palette (adopted in artifact 1 redesign ✅)
- Serif typography for headings (adopted ✅)
- Generous whitespace (adopted ✅)
- Scroll reveal animations (adopted ✅)
- Asymmetric two-column layouts (adopted ✅)
- Dark contrast section for emphasis (adopted ✅)
- Editorial, not template-y

### Portfolio 2: Albert Hong — UX/UI Designer

**What makes it great:**
- Clean white background, zero clutter
- Simple nav: name left, Home/About/Contact right, underline on active page
- Hero: large serif tagline describing what he does, LinkedIn link
- Project grid: 2-column, device mockups (phones, laptops, tablets) on light gray cards
- One-line description under each project card
- Case study structure: centered title → Solution heading → paragraph → device mockups → Outcomes
- Bold inline stats ("conversions up 36%", "calls dropped 42%")
- "Read more of my case studies" cross-linking at bottom
- About page: photo left, bio right, hobbies section with personal artwork
- Contact CTA: bold question + email, clean line
- Footer repeats nav + LinkedIn

**Design takeaways to use:**
- Device mockups make projects feel real
- Structured case study format (problem → solution → screens → outcomes)
- Measurable outcomes with bold stats
- Cross-linking between projects
- Personal touch (hobbies section humanizes)
- Simple, scannable layout
