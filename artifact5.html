What a decision tree looks like

Each internal node asks a yes-or-no question on one feature. Each branch is an answer. Each leaf is a prediction (a class label in classification, or a number in regression).

Plain language

Questions, not magic

Training finds good questions to ask about your inputs — thresholds on numeric features, or branches for categories — so that each leaf ends up with examples that mostly agree on the target label.

Because the path from root to leaf is a list of rules, you can explain a single prediction to a stakeholder without opening a thousand-dimensional weight matrix.

About this project

Structured per the course artifact template (title in the hero; remaining items in this grid).

Introduction Decision trees are one of the most approachable models in supervised learning: they encode decisions as nested rules. This artifact introduces their structure, shows a visual diagram and a hands-on path through a toy problem, and connects the topic to portfolio expectations for AIML coursework. Description The page combines narrative explanation, two static illustrations (tree topology and data-to-prediction flow), a screenshot of the official portfolio requirements, and a small interactive demo that simulates traversing a binary tree. Together they show how classification can be presented without equations while still remaining precise. Objective Help a reader with minimal ML background understand what a decision tree is, how a prediction is produced by following branches, and why interpretability matters — while documenting the work in the format required for the final digital portfolio. Process Mapped course rubric headings to page sections; drafted copy aligned with standard treatments of binary splits and leaves; designed SVG figures for layout-independent “pictures”; implemented the walkthrough as a small state machine in JavaScript; validated accessibility (alt text, aria-live on path log). Tools and technologies used HTML5, CSS (layout, typography, responsive breakpoints), JavaScript (DOM updates for the demo), SVG for diagrams, and static PNG for the course requirements figure. Value proposition Delivers a self-contained teaching artifact: visuals plus interaction in one static page, suitable for sharing with instructors or employers without extra tooling. Unique value The interactive path mirrors real tree inference step-by-step, and the portfolio-requirements figure explicitly ties the page to the program’s artifact checklist — uncommon in generic tutorials. Relevance Decision trees and their ensembles (random forest, gradient boosting) appear across industry tabular ML, credit risk, churn, and fault triage. Interpretable baselines are also increasingly important next to black-box models for audit and trust. References Breiman, L., Friedman, J., Olshen, R., & Stone, C. (1984). Classification and Regression Trees. Chapman & Hall. Mitchell, T. M. (1997). Machine Learning, Chapter 3. scikit-learn documentation: Decision Trees. Course materials: AIML-500, Indiana Wesleyan University (portfolio requirements).
